### Better Models: Worse Tools

**Source:** Simon Willison's Weblog (link-post to Armin Ronacher) · **Published:** 2026-07-04 · **Read time:** ~5 min

**Elevator pitch:** Claude Opus 4.8 and Sonnet 5 reliably invent undeclared fields in tool call schemas when operating in long agentic sessions — a regression that older models do not exhibit and that production benchmarks do not surface.

**Opinion vs. fact:**
- [FACT] Claude Opus 4.8 and Sonnet 5 produce tool calls with extra, invented fields in Pi's edit tool's nested `edits[]` array; Pi rejects the call and retries
- [FACT] The failure is context-dependent: reproduces in long agentic histories (file reads → diagnostics → edits) but not in fresh single-turn prompts
- [FACT] The edit intent is usually correct; the schema mismatch is the proximate failure
- [FACT] Older Claude models do not exhibit this behavior in the same setup
- [OPINION] Root cause is a training artifact: newer models have strong priors shaped by Claude Code's edit schema (flat `old`/`new` string pair plus optional `replace_all`), which they project onto other tools with overlapping structure
- [OPINION] This class of regression will affect any agent using custom tools whose schemas resemble common training-data harness patterns

**References:** Armin Ronacher (original discovery, lucumr.pocoo.org); Simon Willison (commentary and link-post). No studies or data papers cited.

**Justifications:** The context-dependency is the strongest supporting evidence — it rules out random noise and supports the training-prior hypothesis, since longer agentic sessions increase the effective weight of prior patterns. The Claude Code schema match (old/new/replace_all projecting onto Pi's edit structure) is a specific, falsifiable prediction. Weakest point: the training hypothesis is inference; Anthropic has not confirmed the mechanism.

**Core points:**
- Opus 4.8 and Sonnet 5 invent fields consistent with Claude Code's own edit schema when calling third-party tools in agentic contexts
- Failure is context-dependent: grows more likely as session history lengthens (more reads, more prior tool calls)
- The correct edit intent is generated; schema rejection causes retry loops, not silent data corruption
- No older Claude models showed this behavior in the reported setup
- Model upgrades can silently regress production agents at the schema-validation layer, not at the capability layer

**Impact:** This is the failure mode that standard capability benchmarks miss: a model can pass coding evals while breaking the schema contract that makes agents autonomous. For anyone building agentic systems on Anthropic models — especially multi-step pipelines with custom tool schemas — this argues for schema-validation regression tests as a first-class part of model upgrade checklists. The failure pattern (long sessions, tools with structure resembling known harnesses) maps closely to what network orchestration and OSS automation agents look like in practice. In a telecom context the stakes are higher: tool-call failures in provisioning or config-management agents produce retry loops at best and silent partial operations at worst.
