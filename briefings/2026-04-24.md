**Justifications:** This is Anthropic's own disclosure, the highest-confidence source available. Three separate root causes with distinct dates and fixes are independently verifiable against Claude Code changelogs. The explanation of why detection was difficult — overlapping changes affecting different traffic on different schedules — is internally consistent and plausible. The postmortem does not quantify how many users were affected by each issue.

**Core points**
- Two months of degradation came from three separate harness-layer changes, not model regressions
- Reasoning effort silently downgraded to medium (March 4); restored to xhigh/high after user feedback (April 7)
- Context-clearing bug made every session appear idle on every turn, causing apparent forgetfulness (March 26–April 10)

If you're integrating the Claude Agent SDK into CI/CD or deployment pipelines, version-pin your harness configurations and monitor changelogs independently of model releases. The transparency of Anthropic's postmortem is a positive signal for enterprise confidence; the fact that three separate changes each individually slipped through production suggests harness-layer testing is an area to watch.
