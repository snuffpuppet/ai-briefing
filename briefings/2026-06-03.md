### 1. Meta's AI support bot handed over Instagram accounts on request

**Source:** Simon Willison's Weblog · Published: 2026-06-01 · Read time: ~4 min

**Elevator pitch:** Hackers obtained access to high-profile Instagram accounts by asking Meta's AI support chatbot to link a new email address to target accounts — no exploitation required, only the bot's willingness to accept unverified session-context identity claims.

**Opinion vs. fact**
- [FACT] Meta's AI support chatbot accepted a chat-session identity claim and used it to trigger an account recovery flow
- [FACT] Attack chain: claim ownership in chat → receive bot-sent verification code → share code with bot → full password reset
- [FACT] Affected accounts include the Obama-era White House Instagram and a U.S. Space Force account
- [FACT] Instagram confirmed the vulnerability was fixed on June 2, 2026
- [FACT] Per Krebs on Security: accounts with any MFA enabled were not vulnerable to this attack
- [OPINION] The design failure — an agent with elevated privileges accepting unverified context-window identity claims — is not specific to Meta and applies to any AI support agent with account-management authority

**References:** 404 Media (primary reporting with video evidence), Krebs on Security (MFA analysis), Jane Wong (victim/security researcher), Simon Willison commentary

**Justifications:** The attack is documented with hacker-published video. Multiple independent security researchers confirmed it; several were victims. Meta's fix and statement corroborate the vulnerability. Krebs's MFA claim is informed by understanding the attack chain and is credible. The broader generalization about AI agents with elevated privileges is Willison's opinion, not an independently verified claim.

**Core points**
- The bot accepted a false identity claim made within its own context window — no out-of-band verification was performed at any point
- The verification code exchange gave attackers the same privileges as a legitimate account recovery flow; the AI did not distinguish real from false claimants
- Zero technical exploitation: no CVE, no injection into backend systems, no system vulnerability — the AI's lack of identity skepticism was the entire attack surface
- Accounts without MFA were fully exposed; any MFA layer (even SMS) completely blocked the attack
- Meta patched within ~48 hours; the underlying design pattern (AI agent + elevated privileges + no identity verification) persists across the industry

**Impact:** This is the concrete case study for AI agents with account-management or provisioning authority. The design flaw is: elevated privileges granted to an agent that accepts session-context identity claims without independent verification. This class of failure is not exotic — any conversational agent that can modify accounts, tokens, or access permissions based on conversational claims is vulnerable. The correct architecture is explicit: privilege operations triggered by agents must involve out-of-band identity verification, never context-window trust alone. MFA is the minimum floor.

**Impact:** Any team budgeting AI coding tooling needs to model actual token consumption before month-end surprises. The pricing change also creates a forcing function to actually measure what AI coding tooling costs per task — a measurement most teams have been avoiding.
