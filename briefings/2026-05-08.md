### 5. Thoughtworks: taking agentic AI from idea to production

**Title:** The agentic journey from idea to product
**Source:** Thoughtworks Insights · Published: 2026-05-08 · Read time: ~6 min

**Elevator pitch:** A practitioner walkthrough of the architectural decisions, security failure modes, and governance requirements for moving agentic AI from prototype to production in enterprise contexts.

**Opinion vs. fact:**
- [FACT] Gartner cited: up to 40% of agentic AI projects may be canceled by 2027 (Gartner prediction, not Thoughtworks data)
- [FACT] Article identifies a concrete security risk pattern: internal data access + code execution + open internet access = data exfiltration path
- [OPINION] Data platform maturity is a prerequisite for successful agentic AI
- [OPINION] MCP is the right architecture for centralizing agent auth, authz, and audit logging
- [OPINION] Governance and safety should be codified into architecture and delivery pipeline

**References:** Thoughtworks practitioners, Gartner (cited)

**Justifications:** The security risk trifecta is the most specific and actionable claim — directly applicable to permission scoping in agent design. The data-governance-prerequisite argument is well-reasoned and practically grounded. The Gartner cancellation statistic is a citation, not primary evidence, and should be weighted accordingly.

**Core points:**
- Security trifecta: avoid giving a single agent all three of — internal data access, code execution ability, open internet access; any two creates acceptable risk, all three creates a data exfiltration path
- MCP centralizes authentication, authorization, and immutable audit logging across agent-tool interactions
- Governance should be architectural, not bolted on: policy enforcement at the infrastructure layer, not the application layer
- Data governance maturity must precede agentic AI deployment; agents inherit the reliability of their data sources
- Production agents need transparent reasoning logs and human-in-the-loop gates for high-risk actions

**Impact:** Practical design checklist for anyone building agentic systems for enterprise deployment. The security trifecta is a memorable, applicable heuristic for scoping agent permissions — use it to scope-bound any agent you design. The data governance prerequisite is especially relevant for telecom OSS/BSS environments where data provenance and quality vary across systems: an agent relying on inconsistent or unvalidated OSS data will produce unpredictable outputs that erode stakeholder trust faster than they add value. Both the security and data-quality points argue for starting agent integrations with read-only, narrow-scope access and expanding only after reliability is established.
