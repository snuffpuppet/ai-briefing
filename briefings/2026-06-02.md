### 1. Hackers Simply Asked Meta AI to Give Them Access to High-Profile Instagram Accounts. It Worked

Source: Ars Technica / 404 Media · Published: 2026-06-01 · Read time: ~4 min

**Elevator pitch** — Meta's AI support chatbot allowed account takeover with no technical exploit: attackers asked the bot to link a new email to the target's account, confirmed the code sent to their own inbox, and reset the password.

**Opinion vs. fact**

- [FACT] Attackers initiated conversations with Meta AI Support Assistant, claimed to be the account owner, and provided the target's username.
- [FACT] The chatbot dispatched a verification code to the attacker's supplied email address — not the account's registered email.
- [FACT] Confirmed takeovers include the Barack Obama White House account, Space Force Chief Master Sergeant's account, and Sephora's account.
- [FACT] Instagram accounts without multi-factor authentication were most vulnerable.
- [FACT] Meta stated the issue was patched as of June 1, 2026 (Andy Stone, Instagram spokesperson).
- [OPINION] This represents a fundamental design flaw — the AI treated the conversation participant as the authenticated account owner without independent verification — not a one-off implementation bug.

**References** — 404 Media (original reporting), Ars Technica, TechCrunch, Simon Willison's Weblog (secondary analysis); Meta spokesperson Andy Stone confirmed patch via post reply.

**Justifications** — The attack mechanism is confirmed via video evidence showing the chatbot interaction step-by-step. The strongest evidence is the documented, verifiable takeover of named high-profile accounts with no technical exploit involved. The weakest element is the absence of detail on what exactly Meta changed in the patch — whether the fix is structural (identity verification before account mutations) or a surface-level filter. Multiple independent sources corroborate the mechanism.

**Core points**

- Attack required no access to victim's email, no phishing link, no malware — only a natural-language request to the support bot.
- Attackers used a VPN to spoof the victim's presumed location, avoiding automated fraud detection.
- The chatbot sent a verification code to the attacker's email, which the attacker then returned to the bot to complete the "reset password" flow.
- MFA prevented account takeover; accounts without MFA were fully compromised.
- Meta has patched the issue, but the window was sufficient for high-value account handles to be stolen and resold.

**Impact** — This is the clearest recent production demonstration of the failure mode that occurs when an AI agent is granted write access to consequential account operations without enforcing independent identity verification. The exploit required nothing other than a plausible claim in natural language. For anyone designing AI agents with access to account management, configuration changes, or any irreversible action: this case establishes a non-negotiable design requirement — consequential actions must verify identity through a channel the AI does not control. The pattern failure is not prompt injection; it is unconditional trust in conversational context. Every agentic system taking irreversible actions needs an explicit answer to: "What is my ground truth for identity, and can the agent be tricked into substituting its own?"

**Elevator pitch** — NVIDIA released Cosmos 3, an open foundation model trained on 20 trillion tokens that natively reasons about physical scenes and generates synchronized outputs across video, image, audio, text, and robot action data in a single architecture.

**Opinion vs. fact**

- [FACT] Cosmos 3 uses a mixture-of-transformers architecture combining a reasoning block with a generation block.
- [FACT] Trained on 20 trillion tokens including approximately 1 billion images, 400 million real and synthetic videos, audio, text, and robot action data.
- [FACT] Two variants released immediately: "super" (high physics accuracy for robot/AV training) and "nano" (sub-second generation).
- [FACT] Model produces numerical robot action outputs directly — joint angles, gripper positions, trajectory points.
- [FACT] Model weights are open; an "edge" variant for local deployment is forthcoming.
- [OPINION] NVIDIA describes this as "the first open omni-model for physical AI" — the "first" designation has not been independently verified against all existing open models.

**References** — NVIDIA Newsroom, NVIDIA Technical Blog (developer.nvidia.com), Hugging Face blog (NVIDIA authored), HPCwire, Axios.

**Justifications** — The training data figures (20T tokens, 1B images, 400M videos) are stated by NVIDIA. Token-count comparisons across multimodal models are not standardized, so the scale claim should be read as directional. The native action generation capability is the technically distinctive claim — generating numerically grounded robot commands from scene understanding in a single pass is genuinely novel for an open model. The "first" claim is contested ground.

**Core points**

- Reasoning block interprets the scene; generation block produces physically grounded outputs (video, audio, robot actions) from that context — a single unified pass.
- Native action generation: outputs joint angles, gripper positions, trajectory points — not natural-language descriptions of what a robot should do.
- Can generate synthetic training data for rare or dangerous scenarios (collisions, unusual road events) that are costly or unsafe to collect in the physical world.
- Available now: super (high accuracy) and nano (speed). Edge model (local deployment, on-device) coming.
- Open weights — usable without NVIDIA cloud infrastructure.

**Impact** — The primary relevance for AI system designers is synthetic data generation for physical or sensor-rich environments. If your system involves inspection, autonomous navigation, or sensor-fusion at the edge — whether for network equipment, vehicles, or industrial sites — Cosmos 3 reduces the cost of training data for rare failure modes. The open weights matter specifically for data-locality-constrained deployments. For non-physical-AI workloads the direct relevance is lower, but the unified perception-reasoning-action pattern in the architecture signals a direction for multimodal agentic design that goes beyond text + tool calls.

### 4. Open and Closed Models Are on Different Exponentials

Source: Interconnects AI (Nathan Lambert) · Published: 2026-06-01 · Read time: ~8 min

**Elevator pitch** — Nathan Lambert argues that open and closed AI models are diverging economically and technically, with agentic coding being the first domain where sustained willingness to pay frontier model premiums is clearly demonstrated.

**Opinion vs. fact**

- [FACT] Open models lag state-of-the-art closed models by approximately 4 months, per Epoch AI data (cited).
- [FACT] Coding agent tools (Copilot, Cursor, and comparable) are generating sustained revenue at premium pricing.
- [OPINION] The open and closed model economies are on "different exponentials" — Lambert's framing of divergence rather than convergence.
- [OPINION] Through 2026, closed model performance improvements will outpace what open model releases can replicate at speed.
- [OPINION] The primary driver of this divergence is economic, not just technical — users in specific high-value domains are paying significantly more for marginal intelligence gains.

**References** — Epoch AI open-closed model capability gap data; market observations from Interconnects AI.

**Justifications** — The Epoch AI 4-month lag estimate is the strongest factual anchor. The argument about economic divergence rests on inference from observed market behavior (coding agent pricing, enterprise contract values) rather than disclosed financial data. The "exponentials" framing is a strong rhetorical claim that simplifies: in some specific benchmarks, open models have closed the gap faster than predicted. The counterargument — that open models will eventually catch up in capability and undercut closed providers on price — is credible but underspecified in terms of timeline.

**Core points**

- The 4-month capability lag between open and closed models has been approximately stable — convergence is not happening at the rate open-model advocates expected.
- Coding agents are the first demonstrated high-volume market where users accept frontier model pricing premiums — this validates the closed model economic case.
- Binary task domains (code works or doesn't; test passes or fails) favor frontier models because the cost of failure is high and quantifiable.
- For "good enough" domains — classification, retrieval, summarization — open models deliver better total cost of ownership.
- The open/closed choice is framed as a domain-specific economic question, not a one-size-fits-all technical preference.

**Impact** — This piece provides a useful decision framework for vendor and model strategy. For a solutions architect building AI-integrated systems, the practical takeaway is: scope the domain first, then select the model tier. For multi-step agentic workflows where task completion is binary (network configuration, code generation, structured data extraction), the frontier model premium is likely justified. For batch workloads where throughput and cost dominate, open or smaller models on-premises are the stronger choice. The piece also surfaces a risk: as closed models widen capability leads in high-value domains, lock-in risk grows. Architectural patterns that make model substitution tractable (standardized tool interfaces, provider-agnostic APIs) are worth investing in now.

- [OPINION] The change effectively reduces the included value of existing plans for power users.

**References** — GitHub Blog (official announcement), TechCrunch ("What a joke: GitHub Copilot's new token-based billing spurs consternation among devs"), Visual Studio Magazine, TechTimes, GitHub community discussion #192948.

**Justifications** — Community reaction (400+ comments, ~900 downvotes) is documented and strongly negative. The 10-50x cost increase for agentic users is mechanically plausible: agentic sessions consume input + output + cached tokens at listed API rates, and long-context agentic passes are expensive at those rates. Single-day credit exhaustion reports appear from multiple independent users. GitHub's counter-argument — that the change only materially affects unusually heavy users — is not supported by the volume and specificity of complaints. The plan-level credit allowances equal plan price in dollars, which is explicit in the announcement.

**Core points**

- Token consumption counts input, output, and cached tokens at listed API rates — long-context agentic sessions are disproportionately expensive under this model.
- The fallback model has been removed; there is no cheaper in-plan option when credits run low.
- Heavy Copilot users running autonomous coding agents should expect significant cost increases and need to model usage before the end of the billing period.
- Code completions and Next Edit remain flat — inline assistance is unaffected for typical usage.
- The new model aligns Copilot pricing with direct API costs, removing the previously generous flat-rate packaging for agentic workloads.

**Impact** — A direct cost decision point for any team running AI-assisted development at scale. If your team uses Copilot primarily for inline completions, the change is likely immaterial. If your team runs autonomous coding agents — multi-step refactoring, test generation, codebase-wide changes — model your monthly token consumption against the credit allowance before next billing cycle. For enterprise procurement: this makes Copilot TCO harder to predict and potentially more expensive than direct API access (OpenAI on AWS, Anthropic API) for heavy agentic use cases. Worth reassessing Copilot vs. direct API contracts for teams with significant agentic automation workloads.
