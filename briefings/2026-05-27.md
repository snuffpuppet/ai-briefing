### 1. Millions of AI agents imperiled by critical vulnerability in open source package

Source: AI — Ars Technica · Published: 2026-05-26 · Read time: ~5 min

**Elevator pitch:** A host-header injection flaw in Starlette (CVE-2026-48710, "BadHost") lets unauthenticated attackers manipulate URL reconstruction across the entire Python async AI stack.

**Opinion vs. fact:**
- [FACT] CVE-2026-48710 is a critical vulnerability in Starlette, a Python async framework with approximately 325 million weekly downloads.
- [FACT] Starlette reconstructs request URLs by concatenating the HTTP Host header with the request path without first validating the Host header.
- [FACT] Injecting characters such as `/`, `?`, or `#` into the Host header can alter where path boundaries fall in the reconstructed URL, enabling routing manipulation and auth bypass.
- [FACT] The vulnerability cascades through FastAPI into vLLM, LiteLLM, Text Generation Inference, OpenAI-shim proxies, MCP servers, agent harnesses, eval dashboards, and model-management UIs.
- [FACT] Starlette ≥ 1.0.1 patches the vulnerability; a free scanner is available at badhost.org.
- [OPINION] Because Starlette underpins FastAPI, which underpins a large fraction of Python AI tooling, a significant portion of production AI services built in Python are affected.

**References:** OSTIF.org security disclosure; badhost.org CVE tracker. No specific researchers named in the Ars Technica piece.

**Justifications:**
- *Strongest:* The technical mechanism is precisely specified — Host header injection is a well-understood vulnerability class, and the URL reconstruction pattern described is directly exploitable.
- *Weakest:* The blast-radius claim requires inference: not every FastAPI service uses the vulnerable code path, and exploitability in a given deployment depends on routing configuration.

**Core points:**
- Starlette uses an attacker-controllable HTTP Host header to reconstruct request URLs without validation, enabling path manipulation.
- FastAPI inherits the flaw, extending it to the entire Python AI services ecosystem built on both.
- Primary attack vectors are authentication bypass and request routing manipulation.
- Fix is a version bump to Starlette ≥ 1.0.1; badhost.org provides a scanner for identifying affected deployments.
- The vulnerability was publicly disclosed 2026-05-26; exploitation risk is elevated with a public PoC now in circulation.

**Impact:** For any SA running Python-based AI services — which now covers most LLM inference proxies, MCP gateway servers, and agent orchestration frameworks — this is a patch-now item. Check your Starlette version if your stack includes FastAPI. The specific concern for agentic architectures is that MCP server harnesses and eval dashboards are frequently exposed to semi-trusted traffic, making a host-header attack a plausible route to redirecting agent tool calls or manipulating routing in ways that are difficult to surface in logs.

### 2. Microsoft Copilot Cowork Exfiltrates Files

Source: Simon Willison's Weblog · Published: 2026-05-26 · Read time: ~4 min

**Elevator pitch:** Researchers found two independent exfiltration paths in Microsoft Copilot Cowork: one via rendered external images in agent-sent emails, and one exploiting the Anthropic API endpoint being on the product's own domain allowlist.

**Opinion vs. fact:**
- [FACT] Copilot Cowork allows agents to send emails to the user's inbox without explicit approval for each send.
- [FACT] Those emails can contain external image tags; when rendered, the image requests leak data to an attacker-controlled server.
- [FACT] Cowork restricts outbound HTTP traffic to a domain allowlist intended to prevent exfiltration via prompt injection.
- [FACT] Prompt Armor discovered that api.anthropic.com is on that allowlist, enabling an attacker to supply their own Anthropic API key and have the agent upload accessible files to the Anthropic Files API endpoint for later retrieval.
- [OPINION] Simon Willison characterises this as evidence that preventing exfiltration under adversarial prompt injection remains the hardest unsolved problem in agentic system design.

**References:** Prompt Armor (security research firm, credited). Vulnerability first reported January 2026; Willison's writeup surfaced it in May.

**Justifications:**
- *Strongest:* Both attack paths are mechanically specific and reproducible — rendered external images and Anthropic Files API exfiltration are concrete, demonstrated techniques.
- *Weakest:* No Microsoft response or patch timeline is included; severity in patched vs. unpatched state is unclear.

**Core points:**
- Hostname allowlists are insufficient when allowlisted domains can themselves accept and store attacker-controlled data — the Anthropic Files API is a straightforward example, but the class includes any cloud API with persistent storage.
- The rendered-image exfiltration path is a persistent vulnerability class across all email-rendering agents; the fix requires stripping external images or sandboxing render contexts.
- Both attack chains originate from prompt injection; untrusted content processed by agents must be treated as an adversarial boundary throughout.
- Outbound traffic controls for agents require capability-level reasoning about what an endpoint can do with data, not just hostname matching.
- No confirmed patch timeline from Microsoft at time of publication.

**Impact:** Directly relevant for anyone designing or procuring agentic automation over files, email, or document workflows. The Anthropic Files API exfiltration path is a concrete instance of a broad class: any allowlisted domain that provides persistent storage is a potential exfiltration vector. SA-level threat models for agentic systems need to account for this. The rendered-image path reinforces a complementary requirement: agent output containing HTML or markdown must be treated as untrusted and rendered in isolation from the user's network context.
